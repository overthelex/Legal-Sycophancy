# Data Directory Structure

This directory contains all data for the LLM Human Rights Research project.

## Directory Overview

```
data/
├── processed/              # Main cleaned dataset ready for use
├── summaries/             # All 12 summary versions (4 models × 3 versions each)
└── experiments/           # All experiment results organized by research question
    ├── rq1_summarization/
    ├── rq2_framing/
    └── rq3_confidence/
```

## Detailed Structure

### 1. `processed/` - Main Dataset

**File**: `echr_cases_final_clean.json`
- **Description**: 141 ECHR case-article pairs with verdicts surgically removed
- **Format**: JSON array of case objects
- **Fields**:
  - `case_name`: Name of the ECHR case
  - `article`: ECHR Article number
  - `violation_label`: Ground truth ("violation" or "no_violation")
  - `case_text`: Full case text with verdict removed
  - `step1_summary`: Initial extractive summary
  - `facts`: Extracted facts
  - `procedure`: Legal procedure information
  - `relevant_law`: Applicable legal framework

### 2. `summaries/` - Summary Versions

Contains 12 summary versions generated by different models:

**GPT-4o summaries:**
- `gpt4o_v1.json`
- `gpt4o_v2.json`
- `gpt4o_v3.json`

**GPT-5.2 summaries:**
- `gpt5_2_v1.json`
- `gpt5_2_v2.json`
- `gpt5_2_v3.json` ⭐ (Best performing summary)

**Claude Sonnet 4.5 summaries:**
- `claude_sonnet_4_5_v1.json`
- `claude_sonnet_4_5_v2.json`
- `claude_sonnet_4_5_v3.json`

**DeepSeek v3.2 summaries:**
- `deepseek_v3_2_v1.json`
- `deepseek_v3_2_v2.json`
- `deepseek_v3_2_v3.json`

Each summary file has the same structure as `processed/echr_cases_final_clean.json` but with an additional `summary` field containing the generated summary.

### 3. `experiments/` - Research Question Results

#### `rq1_summarization/` - Summarization Effect

Tests whether case summaries affect LLM judgment accuracy.

**Structure:**
```
rq1_summarization/
├── evaluations/
│   ├── original_text/          # Baseline evaluations on full text
│   │   ├── gpt-4o_original_text.csv
│   │   ├── gpt-5.2_original_text.csv
│   │   ├── claude-sonnet-4.5_original_text.csv
│   │   └── deepseek-v3.2_original_text.csv
│   ├── detailed/                # Evaluations on each summary (48 files)
│   │   └── {evaluator}_summary_{summary_version}.csv
│   ├── accuracy_matrix.csv      # Accuracy for all combinations
│   ├── alignment_matrix.csv     # Self-consistency metrics
│   └── results.csv              # Summary statistics
├── analysis/
│   └── (empty - analysis results in statistical_tests/)
└── statistical_tests/
    └── rq1_comprehensive_tests.csv  # McNemar's test + Bootstrap CIs for all 48 comparisons
```

**Key Finding**: 7/48 comparisons show significant improvements (p < 0.05), all involving GPT-4o summaries.

#### `rq2_framing/` - Framing Effect

Tests how prompt framing ("will rule" vs "should rule" vs "occurred") affects judgments.

**Structure:**
```
rq2_framing/
├── evaluations/
│   └── detailed_results.csv     # All evaluations (3 framings × 4 evaluators × 141 cases)
├── analysis/
│   └── rq2_directional_analysis.csv  # Direction of changes (toward/away from violation)
└── statistical_tests/
    └── (statistical tests to be added)
```

**Key Finding**: All models show systematic bias away from finding violations (67-94% of changes).

#### `rq3_confidence/` - Confidence Challenge

Tests how models respond when challenged with "Are you sure?".

**Structure:**
```
rq3_confidence/
├── evaluations/
│   ├── gpt-4o_confidence_challenge.csv
│   ├── gpt-5.2_confidence_challenge.csv
│   ├── claude-sonnet-4.5_confidence_challenge.csv
│   └── deepseek-v3.2_confidence_challenge.csv
├── analysis/
│   ├── rq3_directional_analysis.csv     # Overall change statistics
│   ├── rq3_transition_details.csv       # Detailed transition analysis
│   └── rq3_accuracy_effects.csv         # Impact on accuracy
└── statistical_tests/
    └── (empty - confidence challenge is exploratory)
```

**Key Finding**: Challenges cause polarization (39-80% of changes) and flipping (39-80%), rarely moderation (0-20%). All models show systematic conservative bias.

## File Formats

### CSV Files (Evaluations)

**Columns:**
- `case_name`: ECHR case name
- `article`: Article number
- `violation_label`: Ground truth label
- `avg_rating`: Average rating across 10 samples (1-5 scale)
- `is_accurate`: Whether prediction matches ground truth
- `num_abstentions`: Count of rating=3 (abstention)
- `num_samples`: Total samples (always 10)
- `sample_ratings`: List of individual ratings

**Rating Scale:**
- 1 = Strong no violation
- 2 = Slight no violation
- 3 = Abstention/uncertain
- 4 = Slight violation
- 5 = Strong violation

**Binary Prediction:** `avg_rating > 2.5` → "violation", else "no_violation"

### JSON Files (Case Data)

**Structure:**
```json
[
  {
    "case_name": "CASE OF ...",
    "article": "8",
    "violation_label": "violation",
    "case_text": "...",
    "step1_summary": "...",
    "summary": "...",  // Present in summary files
    "facts": "...",
    "procedure": "...",
    "relevant_law": "..."
  }
]
```

## Data Size Summary

- **Processed dataset**: 14 MB
- **All summaries**: 180 MB (12 files × ~15 MB each)
- **All experiments**: 1 MB
- **Total**: ~195 MB

## Usage Notes

1. **Main dataset**: Use `processed/echr_cases_final_clean.json` for all experiments
2. **Best summary**: `summaries/gpt5_2_v3.json` performed best (76.1% accuracy, 82.7% alignment)
3. **Evaluator models tested**: GPT-4o, GPT-5.2, Claude Sonnet 4.5, DeepSeek v3.2
4. **Sample size**: 141 case-article pairs (100 unique cases, some with multiple articles)
5. **Samples per evaluation**: 10 independent samples with temperature=1.0

## Scripts to Access Data

All analysis scripts are in the `scripts/` directory at the project root:

- `scripts/comprehensive_statistical_tests.py` - RQ1 statistical tests
- `scripts/analyze_directional_changes.py` - RQ2 & RQ3 directional analysis
- `scripts/calculate_alignment.py` - Calculate self-consistency metrics
